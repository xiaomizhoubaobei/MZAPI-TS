## 常见问题

### 1. 认证失败

**问题描述**：调用API时出现认证失败的错误

**可能原因**：
- API Key不正确或已过期
- 请求头中的认证信息格式错误
- API Key的权限不足

**解决方案**：
1. 检查API Key是否正确填写
2. 确保请求头中包含正确的认证信息
3. 验证API Key的有效性和权限设置

### 2. 请求超时

**问题描述**：API请求响应时间过长或超时

**可能原因**：
- 网络连接不稳定
- 服务器负载过高
- 请求参数过大

**解决方案**：
1. 检查网络连接状况
2. 适当增加超时时间设置
3. 优化请求参数大小
4. 考虑使用重试机制

### 3. 流式输出异常

**问题描述**：使用流式输出时出现断开或数据异常

**可能原因**：
- 网络不稳定导致连接断开
- 客户端处理逻辑错误
- 服务端异常

**解决方案**：
1. 实现自动重连机制
2. 添加错误处理和恢复逻辑
3. 检查客户端代码是否正确处理流式数据

## 错误码说明
错误码是MZAPI在处理请求时返回的状态码，用于指示请求的处理结果和原因。
详细错误码请查看[错误码列表](api.mizhoubaobei.top/error-codes.html)。

## API调用问题

### 请求参数检查

1. 确保所有必需参数都已正确提供
2. 检查参数格式和类型是否符合要求
3. 验证参数值是否在有效范围内

### 响应处理

1. 正确解析响应数据格式
2. 实现适当的错误处理机制
3. 记录关键响应信息用于调试

## 性能优化建议

### 1. 连接管理

- 使用连接池管理API连接
- 实现请求重试机制
- 适当设置超时时间

### 2. 缓存策略

- 合理使用客户端缓存
- 避免重复请求相同数据
- 实现缓存更新机制

### 3. 并发控制

- 控制并发请求数量
- 实现请求队列机制
- 避免过度频繁的API调用
